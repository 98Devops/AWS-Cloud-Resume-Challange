Projects

1. Virtualization Project

Goal: To automate the creation and management of Virtual Machines (VMs) using Vagrant and VirtualBox, simulating a real-world cloud environment.

Why: Virtualization is a foundational skill in cloud engineering. This project demonstrates my ability to automate infrastructure provisioning and manage virtualized environments.

Steps:

  • Used Vagrant to define VM configurations in a Ruby-based Vagrantfile, enabling reproducible and version-controlled infrastructure.
  • Configured static IPs, bridged networking, and resource allocation (RAM, CPU cores) to mimic production environments.
  • Deployed an HTTPD server on the VM and hosted a custom static website, showcasing my ability to manage web servers.
  • Documented the process and created a YouTube tutorial series to share my knowledge with the community.

Skills Demonstrated: VM automation, scripting, server management, networking, and technical communication.

2. AWS Cloud Resume Challenge with Terraform and CI/CD

Goal: To build a serverless resume website hosted on AWS, using Infrastructure as Code (Terraform) and automated CI/CD pipelines.

Why: This project showcases my ability to design and deploy scalable, secure, and maintainable cloud solutions using modern DevOps practices.

Steps:

  • Terraform (IaC): Wrote Terraform scripts to provision AWS resources (S3, CloudFront, Route 53, Lambda, DynamoDB, API Gateway) in a repeatable and version-controlled manner.
  • CI/CD with GitHub Actions: Set up a CI/CD pipeline to automatically deploy changes to the website whenever code is pushed to the GitHub repository.
  • Serverless Backend: Used AWS Lambda and DynamoDB to create a visitor counter, demonstrating my ability to build serverless applications.
  • Security: Implemented IAM roles, S3 bucket policies, and HTTPS to ensure the website is secure and compliant with best practices.

Skills Demonstrated: Cloud architecture, Terraform, CI/CD, serverless computing, and security.
